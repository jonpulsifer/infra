- NFS/PXE and signed native-boot server — **boot-critical for [[Hosts/rackpi5]]** ([[ADR/0008 Diskless netboot for rackpi5]]); monitored by folly.
- Only Pi with NVMe storage (128 GB Patriot P300). Config: `nix/hosts/spore.nix`.
- Reached as `spore.lolwtf.ca`; the Tailscale management UI uses `spore.pirate-musical.ts.net/spore`.
- ## Spore application
	- The read-only network-boot catalog/status app runs directly as `spore.service`; there is no Spore container. Node listens only on loopback and nginx owns network access.
	- Desired iPXE and native-boot policy is generated by Nix. Host MACs come from `terraform/network/unifi/folly/clients.yaml`; policy lives in `nix/hosts/spore.nix`. Change either through a PR, then rebuild the host from merged `main`.
	- PXE clients may read `/spore/api/boot/…` and `/spore/api/scripts/…`; rackpi5 reads `/spore/api/native-boot/rackpi5/{boot.sig,boot.img,nix-store.squashfs}`. The management UI uses a separate Tailscale-authenticated vhost.
	- Runtime observations live in `/var/lib/spore/observations.db`. The database is not authoritative for boot behavior; losing it resets history only. The NixOS backup timer writes retained SQLite backups under `/var/backup/spore`.
	- `spore-native-boot-rackpi5.service` runs as root, verifies the image-embedded squashfs digest, reads `/var/lib/pi-boot-sign/private.pem`, signs `boot.img`, and atomically switches `/var/lib/spore-native-boot/rackpi5`. Digest-addressed roots remain available across activation. nginx serves those files only through its internal acceleration location; the publisher tree is separate from Spore's observation database permissions.
	- `spore.service` and nginx require the native publisher. A Spore application failure does not affect the separate x86 static PXE tree, but it intentionally prevents rackpi5's sole boot decision route.
- ## Build and recovery
	- Validate without activating: `nix build .#nixosConfigurations.spore.config.system.build.toplevel --no-link` on a native aarch64 builder.
	- Deploy from merged Git with the standard NixOS host workflow. Before cutting over rackpi5, verify the publisher unit is active, all three native artifacts return successfully, and the EEPROM contains the matching public key.
	- Roll back Spore with `sudo nixos-rebuild switch --rollback`; rebuilding the older generation atomically republishes its matching signed rackpi5 artifacts.
	- On first migration from the legacy mixed-state database, Spore preserves a recoverable copy before creating the observations-only schema. Restore that backup only for forensic recovery; reviewed Git/Nix catalog state remains authoritative.
	- The pre-start migration is best-effort. If observation storage is corrupt or unavailable, `spore.service` still starts with catalog-backed boot decisions while health remains degraded and observation writes are disabled.
